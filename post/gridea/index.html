Gridea 静态博客写作客户端

2022-11-07

#前言

当下大多数人可能已经对 Github Pages 不再陌生
他是很多喜欢写文章的人第一次接触的用于免费搭建 Blog 的工具
使用 Github Pages 你能很快速的构建一个简洁却又不失优雅的个人 Blog
直到现在互联网上还有大量的基于 Github Pages 的个人博客和搭建方法的教程
然而搭建虽然简单，但是管理和上传文章却相对麻烦不少
官方的教程里，你需要经历繁琐的 Git 步骤才能发布一篇文章和修改你的个人空间界面
对于那些想更便捷和优雅的操作 Github Pages 写作者来说
我郑重推荐这款静态博客写作客户端 Gridea

#回顾

上期:https://blog.huizenx.top/post/web/
我们利用Github开了一个仓库
但是有些人HTML不会写
但Gridea正好可以解决你的烦恼

#食用方法

下载地址:Gridea.Dev
用Github或者Gitee下载
安装完成

#配置

点击远程
我们可以选择各种配置

平台是Github Pages就好
域名处决与你的仓库名称[必须是username+.github.io]
仓库名称和上面一样就行
注意！！！
下面不要按着”master“填
现在Github更新了
从”master“更新为了”main“
邮箱可以不填
令牌最重要了

#令牌

打开这个网址:https://github.com/settings/tokens/new
创造一个新令牌
可以按着我这么填

#配置

CHAME可以不填
这时你可以”检测远程连接“
如果不行可以使用Dev-Sidecar
Dev-Sidecar:https://blog.huizenx.top/post/dev-sidecar/

#后语

学会了Gridea
